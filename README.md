# Finance Dashboard API

Express + TypeScript + Prisma + PostgreSQL: financial records, JWT auth, role-based access (VIEWER / ANALYST / ADMIN), and dashboard aggregates.

## Quick start

1. Copy env: `cp .env.example .env` (Windows: `Copy-Item .env.example .env`)
2. Start PostgreSQL: `docker compose up -d` (Docker Desktop must be running). Default URL uses user `postgres` / password `postgres` and database `finance`. If you still see **role does not exist**, another Postgres may be bound to port 5432—stop it, or run `docker compose down -v && docker compose up -d` once to recreate the Docker volume (clears data).
3. `pnpm install && pnpm prisma generate && pnpm prisma migrate deploy && pnpm prisma db seed`
4. `pnpm dev`

- API root redirects to docs: `http://localhost:3000` → Swagger UI (`/api-docs`). After login, use **Authorize** with the JWT `token` for protected routes.

## Assignment mapping

| Requirement | Implementation |
|-------------|----------------|
| Users & roles | Admin `/users` CRUD; `role`, `status`; seed accounts |
| Records | CRUD + filters (`from`, `to`, `category`, `type`, `q`) + `page`/`limit` + `sort` |
| Dashboard | `GET /dashboard/summary` — totals, `byCategory`, `recentActivity`, `trends` (`granularity=week\|month`) |
| Access control | JWT + role checks; 403 on forbidden routes |
| Validation / errors | Zod + JSON errors; 400/401/403/404/409/500 |
| Persistence | PostgreSQL via Prisma migrations |

## RBAC

| Role | Records | Dashboard | Users |
|------|---------|-----------|-------|
| VIEWER | no | yes | no |
| ANALYST | read | yes | no |
| ADMIN | full | yes | full |

**Viewer `recentActivity`**: `date`, `type`, `category`, `amount` only (no `id`, `notes`, `ownerId`).

## Deploy on Render (Docker)

1. Push this repository to GitHub.
2. In Render, choose **New +** -> **Blueprint** and select this repo.
3. Render will read `render.yaml` and create:
   - web service: `finance-backend` (Docker)
   - PostgreSQL database: `finance-postgres`
4. Wait for first deploy; startup runs `pnpm prisma migrate deploy` automatically in the container.
5. Open your service URL and verify:
   - `/health`
   - `/api-docs`

Notes:
- `JWT_SECRET` is auto-generated by Render from `render.yaml`.
- `DATABASE_URL` is injected from the Render Postgres connection string.
- If you want seed data in production, run a one-off job with `pnpm prisma db seed`.

Use [pnpm](https://pnpm.io/installation); do not commit `package-lock.json`.

## Assumptions

- Single-tenant demo: analysts see all records; admins own new records unless `ownerId` is set.
- Inactive users cannot log in; JWT does not live-revoke on status change until expiry.
